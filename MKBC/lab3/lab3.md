Найдите в разделе 4.2 документа PM0253 функции драйвера CMSIS, предназначенные для работы с приоритетами и запишите в конспект их сигнатуру и назначение.
### 1. `void NVIC_SetPriority(IRQn_Type IRQn, uint32_t priority)`
Устанавливает приоритет для указанного прерывания или исключения (с конфигурируемым уровнем приоритета).  
Параметр `IRQn` – номер прерывания/исключения (тип `IRQn_Type`), параметр `priority` – значение приоритета (в соответствии с текущей группировкой полей приоритета).
### 2. `uint32_t NVIC_GetPriority(IRQn_Type IRQn)`
Возвращает текущий приоритет указанного прерывания или исключения (с конфигурируемым уровнем приоритета).  
Параметр `IRQn` – номер прерывания/исключения. Функция возвращает значение приоритета.

Найдите в разделе 3.2 документа PM0253 функции драйвера CMSIS, предназначенные для работы с регистрами PRIMASK,BASEPRI и FAULTMASK и запишите в конспект их сигнатуру и назначение.
### Регистр **PRIMASK** (маскирует все прерывания с настраиваемым приоритетом, кроме NMI и HardFault)
`uint32_t __get_PRIMASK(void)`  
Возвращает текущее значение регистра PRIMASK.
`void __set_PRIMASK(uint32_t value)`  
Устанавливает новое значение регистра PRIMASK. Значение `1` запрещает обработку всех прерываний (кроме NMI и HardFault), значение `0` разрешает.
### Регистр **BASEPRI** (маскирует прерывания с приоритетом выше заданного порога)
`uint32_t __get_BASEPRI(void)`  
Возвращает текущее значение регистра BASEPRI.
`void __set_BASEPRI(uint32_t value)`  
Устанавливает порог приоритета. Прерывания с приоритетом, численно бóльшим (т.е. более низким приоритетом), чем значение в BASEPRI, маскируются. Значение `0` отключает маскирование.
### Регистр **FAULTMASK** (маскирует все прерывания, кроме NMI и Reset)
`uint32_t __get_FAULTMASK(void)`  
Возвращает текущее значение регистра FAULTMASK.
`void __set_FAULTMASK(uint32_t value)`  
Устанавливает значение регистра FAULTMASK. Значение `1` запрещает обработку всех прерываний (кроме NMI и Reset), значение `0` разрешает.

### Блоки SRAM и FLASH памяти в МК STM32H745 (по документации RM0399, разделы 2.3–2.4)

#### **FLASH-память**
- **Основная FLASH**  
    Адрес: `0x0800 0000` – `0x081F FFFF` (2 Мбайт)  
    Размер: **2 Мбайт** (два банка по 1 Мбайт)
#### **SRAM-память**

| Название    | Адрес                         | Размер     | Примечание                            |
| ----------- | ----------------------------- | ---------- | ------------------------------------- |
| DTCM        | `0x2000 0000` – `0x2001 FFFF` | 128 Кбайт  | Данные с тесной привязкой к ядру (M7) |
| AXI SRAM    | `0x2400 0000` – `0x2407 7FFF` | 480 Кбайт¹ | Общая для обоих ядер, быстрый доступ  |
| SRAM1       | `0x3000 0000` – `0x3001 FFFF` | 128 Кбайт  | Доступна через шину AHB               |
| SRAM2       | `0x3002 0000` – `0x3003 FFFF` | 128 Кбайт  |                                       |
| SRAM3       | `0x3004 0000` – `0x3004 7FFF` | 32 Кбайт   |                                       |
| SRAM4       | `0x3800 0000` – `0x3800 FFFF` | 64 Кбайт   |                                       |
| Backup SRAM | `0x3880 0000` – `0x3880 0FFF` | 4 Кбайт    | Сохраняется при сбросе (питание VBAT) |

### 2. Использование памяти в стандартных скриптах компоновщика CMSIS

Скрипты компоновщика находятся по пути  
`C:\PlatformIO\packages\framework-cmsis-stm32h7\Source\Templates\gcc\linker\`

#### **`stm32h745xx_flash_CM7.ld`** (для ядра Cortex‑M7, прошивка во FLASH)

- **FLASH (ROM)**:  
    Адрес: `0x0800 0000`  
    Размер: обычно весь объём FLASH (2 Мбайт) или выделенная область.  
    Используется для размещения кода и констант.
    
- **RAM (данные)**:  
    Адрес: `0x2000 0000` (DTCM)  
    Размер: `128K`  
    Используется для размещения переменных, стека и кучи.  
    _(В некоторых вариантах скрипта может быть выбран AXI SRAM, но по умолчанию для M7 используют DTCM как наиболее быструю память данных.)_
    

#### **`stm32h745xx_flash_CM4.ld`** (для ядра Cortex‑M4, прошивка во FLASH)

- **FLASH (ROM)**:  
    Адрес: `0x0800 0000` (общая с M7, но области разделяются для каждого ядра)  
    Размер: выделенная часть FLASH (обычно первый или второй банк, зависит от конфигурации).
    
- **RAM (данные)**:  
    Адрес: `0x3000 0000` (SRAM1)  
    Размер: `128K`  
    Используется для данных. SRAM1 доступна как для M7, так и для M4, что позволяет использовать её для обмена данными между ядрами.
    

#### **`stm32h745xx_sram1_CM7.ld`** (для выполнения кода из SRAM)

- **ROM / RAM (код и данные)**:  
    Адрес: `0x3000 0000` (SRAM1)  
    Размер: `128K`  
    Весь код и данные размещаются в SRAM1. Это позволяет выполнять программу из SRAM (например, для энергосбережения или ускорения). В лабораторной работе этот скрипт использовался для запуска приложения из **AXI SRAM**, но в имени файла сохранено «sram1», что требует осторожности: возможно, он был модифицирован для работы с AXI SRAM. По умолчанию же он ориентирован на SRAM1.

# Ответы на вопросы
1. **Прерывание** – внешнее событие, **исключение** – внутреннее событие ядра (ошибка, SysTick и т.п.).
2. **Таблица векторов прерываний** – массив адресов обработчиков, начинается с начального SP и ResetHandler.
3. **MemManageFault** – нарушение защиты памяти, попытка выполнения из запрещённой области.
4. **BusFault** – ошибка шины (например, чтение из несуществующей памяти).
5. **UsageFault** – неопределённая инструкция, деление на ноль, невыровненный доступ.
6. **HardFault** – возникает, если другое исключение не разрешено или произошла вложенная ошибка.
7. Обработчик должен быть объявлен с именем из ТВП, сохранять контекст (автоматически), выполнить действия, затем вернуться (EXC_RETURN).
8. **Маскирование** – блокировка обработки, но не появления. Можно замаскировать все, кроме NMI, Reset, HardFault.
9. Программная активация – установка pending бита в регистре NVIC или SCB->ICSR.
10. Приоритет можно изменить для всех исключений, кроме Reset, NMI, HardFault.
11. **Pending bit** – флаг, что исключение ожидает обработки.
12. Групповой приоритет определяет вытеснение, подгрупповой – порядок среди равных групп. Максимум 8 уровней групп.
13. CMSIS: `__disable_irq()`, `__enable_irq()`, `__set_BASEPRI()`.
14. `NVIC_SetPriority()`, `NVIC_EnableIRQ()`.
15. Загрузчик – запускает основное приложение, обновляет прошивку, проверяет целостность.
16. Загрузчик: отключить прерывания, установить SP приложения, VTOR, переход на ResetHandler.
17. В STM32H745: AXI-SRAM (0x24000000), SRAM1-4 и т.д.
18. Регистр SCB->AIRCR (PRIGROUP).
19. **weak** – позволяет переопределить функцию, если она определена пользователем.
20. При прерывании: сохраняется контекст (8 слов), определяется вектор, переход в ISR.
21. Выполнение из SRAM – для энергосбережения, детерминированного времени, ускорения, перезаписи FLASH.
22. Кадр исключения: PSR, PC, LR, R0-R3, R12.
23. Вход и выход – по 12 тактов.
